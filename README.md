# CloudWalk Recruiting Analysis

This repository contains my analysis of the fictional recruiting dataset provided during the CloudWalk hiring process.

Files

- `Funnel Analysis - Cloudwalk.pdf` → Complete analysis and recommendations
- `mock_sourcing_dataset.xlsx` → Original dataset used
- `Cloudwalk Presentation` → Simplified presentation with essential data

Alloted time: 5 hours
AI Usage: The data set was analysed through Claude by brainstorming and providing insights from myself. Organization/formatting were done through a simple notetaker, and ChatGPT.
Presentation was done with Gamma and edited by myself.

Focus Areas

- Funnel and conversion analysis
- Recruiter performance patterns
- Sourcing channel effectiveness
- Candidate progression signals
- AI-assisted recruiting opportunities

Main Insights

- 32% of candidates did not respond to first outreach
- Github showed the strongest hiring conversion
- Candidates with technical + behavioral scores 80 or higher had ~2x higher hire rate
- The largest funnel losses occur right after technical assessments
- Recruiter specialization patterns emerged across specific roles

Suggestions

Across six months of recruiting activity, the dataset suggests that the operation’s largest opportunity is not pipeline generation, but improving conversion efficiency among already-qualified candidates.

The strongest improvements likely will come from:

- recruiter specialization
- better post-test candidate management
- stronger outreach personalization
- AI-assisted prioritization of high-potential profiles
